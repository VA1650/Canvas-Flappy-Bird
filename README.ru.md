# Canvas Flappy Bird (OOP Arcade Engine)

Микро-аркада на чистом JavaScript, реализованная на базе HTML5 Canvas API без использования сторонних игровых движков. Архитектура построена на паттерне единого стейт-менеджера (`game`), управляющего игровым циклом через нативный механизм синхронизации развертки дисплея.

## 🏗 Архитектурные решения и оптимизация

* **Утечки памяти и Сборка мусора (Garbage Collection):** В базовых реализациях Flappy Bird массивы препятствий бесконечно растут, что приводит к деградации производительности при длительных игровых сессиях. В данном движке реализована автоматическая очистка: как только координата левой границы колонны (`col.x`) уходит за пределы холста с учетом её ширины, объект мгновенно удаляется из начала массива через высокопроизводительный метод `this.columns.shift()`.
* **Асинхронный пакетный предзагрузчик (Asset Preloader):** Для исключения гонки условий (Race Condition) при загрузке текстур реализован кастомный счетчик асинхронных событий `onload`. Игровой цикл и генерация геометрии не запустятся до тех пор, пока все графические ресурсы (`bird`, `background`, `columns`) не будут полностью кэшированы в оперативной памяти и готовы к отрисовке. Это предотвращает падение рендера на медленных соединениях.
* **Синхронизация кадров (Render Loop):** Вместо дискретных таймеров `setInterval`, нагружающих поток выполнения и вызывающих микрофризы, анимация завязана на `requestAnimationFrame`. Это гарантирует плавный рендеринг, синхронизированный с частотой обновления экрана (60Hz / 120Hz / 144Hz) и автоматически приостанавливает вычисления при переключении вкладок браузера, экономя ресурсы процессора.
* **Инвариантный расчет коллизий (AABB Collision):** Алгоритм детекции столкновений обрабатывает пересечение прямоугольных ограничивающих рамок (Axis-Aligned Bounding Box) птички и двух сегментов колонн одновременно, включая валидацию выхода за верхний и нижний триггеры игрового поля.

## 🎨 Стилизация и системный контекст

Интерфейс поддерживает системный медиа-запрос `prefers-color-scheme`. Окружение холста автоматически адаптируется под глобальные настройки операционной системы пользователя:
* **Dark Mode (по умолчанию):** Черное пространство вокруг игрового экрана, контрастная белая рамка холста.
* **Light Mode:** Белое окружение, четкая черная рамка для фокуса на игровом процессе.

## 🔧 Структура файлов проекта

* `index.html` — Декларативная разметка, инициализация холста `<canvas>` фиксированного разрешения (288x512px) и подключение стилей.
* `script.js` — Логика машины состояний, обработка пользовательского ввода (`keydown`), расчет физики гравитации и детекция коллизий.

## 🎮 Управление

* `Любая клавиша на клавиатуре` — Импульс взлета (противодействие силе гравитации).
* При столкновении с препятствием или падении игра автоматически перезапускает контекст сессии (`location.reload()`), обнуляя счетчик очков.
